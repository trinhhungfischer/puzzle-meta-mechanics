# Design Spec: US-032 - Mechanic cards show representative game icons

## Purpose
Make Mechanic cards visually scannable by displaying representative game covers on the `MechanicCard` component. This will provide immediate context about which games utilize the mechanic.

## Approach Selected
**Approach A**: Overlapping circular avatars at the bottom next to the "Used in X games" count.

## Implementation Details

### 1. Data Fetching
Modify the query in `src/app/mechanics/page.tsx` to include the `games` relation inside `mechanics`.
We will fetch a subset (e.g., up to 4) of `games` (published only) that use this mechanic. We need `coverUrl` and `title` from the associated `game`.

```typescript
mechanics: {
  include: {
    _count: { select: { games: true } },
    games: {
      take: 4,
      where: { game: { status: 'published' } },
      select: {
        game: {
          select: { title: true, coverUrl: true }
        }
      }
    }
  }
}
```

### 2. Component Update
Update `MechanicCardProps` in `src/components/ui/MechanicCard.tsx` to reflect the fetched games data:

```typescript
type MechanicCardProps = {
  mechanic: {
    id: string
    slug: string
    name: string
    description?: string | null
    games?: { game?: { title: string, coverUrl: string | null } }[]
    _count?: { games: number }
  }
}
```

### 3. UI Presentation
In `src/components/ui/MechanicCard.tsx`, modify the footer section:
- Map over `mechanic.games` and render overlapping `<img>` tags (circular, `rounded-full`, with a border).
- If `coverUrl` is missing, render a fallback colored circle using the first letter of the game's title.
- Render these avatars adjacent to the "Used in X games" count text.

Example structure:
```tsx
<div className="mt-auto pt-4 border-t border-white/5 flex items-center gap-3">
  <div className="flex -space-x-2">
    {mechanic.games?.map((gm, i) => gm.game && (
      gm.game.coverUrl ? (
        <img key={i} src={gm.game.coverUrl} title={gm.game.title} className="w-6 h-6 rounded-full border border-zinc-900 object-cover" />
      ) : (
        <div key={i} title={gm.game.title} className="w-6 h-6 rounded-full border border-zinc-900 bg-brand-violet/50 flex items-center justify-center text-[10px] font-bold text-white uppercase">
          {gm.game.title.charAt(0)}
        </div>
      )
    ))}
  </div>
  <div className="text-xs font-medium text-zinc-500">
    Used in {gameCount} game{gameCount !== 1 ? 's' : ''}
  </div>
</div>
```

## Considerations
- Fallbacks are provided in case a game has no cover image yet.
- The use of `next/image` is deferred until US-038 (Performance - Optimize cover images with next/image), so standard `<img>` tags are used for now to keep scope constrained to presentation only.
